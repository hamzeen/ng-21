## Accessibility Checklist

### Keyboard Navigation

- [ ] All interactive elements reachable via Tab
- [ ] Focus visible and clear
- [ ] Tab order logical
- [ ] No keyboard traps
- [ ] Escape closes dialogs

### Screen Readers

- [ ] All content accessible to screen readers
- [ ] Form labels associated with inputs
- [ ] ARIA labels on icon buttons
- [ ] Page structure with semantic HTML
- [ ] Live regions for dynamic content

### Color & Contrast

- [ ] Text contrast ≥ 4.5:1 (normal text)
- [ ] Large text contrast ≥ 3:1
- [ ] Color not sole indicator of state
- [ ] Icons and patterns support color meaning

### Forms & Labels

- [ ] All inputs have labels
- [ ] Error messages clear and associated
- [ ] Required fields marked
- [ ] Input types match content (email, number, etc.)

### Images & Media

- [ ] Descriptive alt text on all images
- [ ] Decorative images have empty alt
- [ ] Videos have captions or transcripts

## Run Accessibility Check

Manual verification:

1. Navigate with keyboard only (no mouse)
2. Use screen reader (e.g., NVDA, JAWS, VoiceOver)
3. Check contrast ratios with DevTools
4. Verify focus indicators visible

Automated tools:

- **axe DevTools** Chrome extension
- **Lighthouse** in Chrome DevTools (Accessibility audit)
- **WAVE** browser extension

## Common Issues

- Missing alt text on images
- Insufficient color contrast
- Form inputs without labels
- Interactive elements not keyboard accessible
- Missing ARIA labels on icon buttons

## WCAG 2.1 AA Standards

All components must pass:

- Keyboard accessibility (WCAG 2.1 2.1.1)
- Focus visible (WCAG 2.1 2.4.7)
- Color contrast (WCAG 2.1 1.4.3)
- Labels and instructions (WCAG 2.1 3.3.2)
- Meaningful sequence (WCAG 2.1 1.3.2)
