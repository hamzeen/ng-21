# Example 3: Login Form Workflow

## Step 1 — Spec

```md
/spec Build a login form with username and password fields, validation messages, disabled submit when invalid, and a fake successful login state.
```

---

## Step 2 — Plan

```md
/plan Use Angular 21 reactive forms, standalone components, typed form controls, and Vitest.
Do not write production code yet.
```

---

## Step 3 — TDD

```md
/tdd Write failing tests for:

- rendering username and password fields
- showing validation errors
- disabling submit when invalid
- enabling submit when valid
- showing success message after submit
```

---

## Step 4 — Implement

```md
Now implement the login form using Angular 21 reactive forms.
Keep validation logic simple and colocated with the component unless reuse is needed.
```

---

## Step 5 — Review

```md
/review Review the login form for Angular best practices, accessibility, validation correctness, and test quality.
```

---

## Step 6 — Ship

```md
/ship Confirm this login form is ready to merge.
Include final checks, risks, and a suggested commit message.
```
